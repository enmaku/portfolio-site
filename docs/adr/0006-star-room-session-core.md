# Star-room session core (Timer and Movie Vote)

Issue #117 extracts shared host/guest wire ceremony from Game Timer and Movie Vote into `createStarRoomSession` (name TBD) under `src/features/p2p/`, building on the existing **star-room shell** reconnect loops and the RTDB factory from #116 / ADR 0005. Feature `session.js` files remain the public facade—same exports and import paths; the core is store-agnostic (no Pinia/Quasar, no Vue refs). Connection posture flows through callbacks (`onPhaseChange`, `onSuffixChange`, …); facades mirror into existing `sessionPhase` / `sessionSuffix` refs. Persistence and domain handling are injected the same way. User-visible messaging uses structured `onSessionEvent` payloads (e.g. `host_ended_room`, `reconnect_attempt`)—wording and Quasar `Notify` stay in feature facades, not in core tests.

Delivery is phased: slice 1 moves host-ping helpers, `isRoomMarkedEnded`, and unified **host reclaim** (`isReclaimOwnHostRoom`—Movie Vote’s stricter ping requirement was a bug); slice 2 introduces the factory. Domain wire formats stay in per-**project** **protocol adapters** (`parseHostInbox`, `encodeGuestHello`, etc.) passed into the core—the factory does not import Game Timer or Movie Vote `protocol.js`. Low-level host behavior is unified where products had drifted: **host abrupt disconnect** (`onDisconnect` clears `hostPing` and sets `ended` for both projects), **host occupancy guard** on every host finish path (including Movie Vote resume), and reclaim without requiring a fresh ping. Guest behavior is not unified: `guestPresence: 'loose' | 'strict'` selects **loose guest attachment** (Game Timer—broadcast-oriented, informational `hostPing`) vs **strict guest presence** (Movie Vote—quorum needs live counts; core maintains **guest online signal** plus ping-null teardown). Per-project `claimResetPaths` and protocol encode/decode stay in feature modules.

**Considered options:** Single PR for the full extraction (rejected—review risk). Pinia-aware core (rejected—testability and #118 coupling). One guest wire policy for all projects (rejected—Movie Vote quorum vs Game Timer drop-in guests). Feature-only `onDisconnect` for Movie Vote (rejected—crash signaling should be shared).

**Consequences:** Tests concentrate host-ping/reclaim/occupancy in `p2p/`; feature tests keep one representative flow each. A third star-room **project** should bind the factory with an explicit presence mode rather than copying another `session.js`. Dungeon Runner stays out of scope (no star-room **room**). Movie Vote Pinia bridge remains #118.
